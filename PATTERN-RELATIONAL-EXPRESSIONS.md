# Pattern Dynamics Relational Expressions

This document describes the **relational expression system** for Pattern Dynamics holarchies, based on comprehensive study of the markdown documentation in the `patterns/` folder. The system captures the 49 relationships in the 7x7 pattern matrix and the holarchical structures that organize these patterns.

## Seven Core Patterns (First Order)

The foundation of Pattern Dynamics consists of seven fundamental patterns:

1. **Source** - The nondual ground from which all patterns emerge

## Holarchical Relationships

Patterns are organized in holarchical levels, where higher levels **transcend and include** lower levels.

### Holarchy Type System

Seven types of holarchical relationships:

1. **Contains** - Parent contains child as component
2. **Transcends** - Higher level transcends lower while including it
3. **Includes** - Includes as part of a larger whole
4. **Emerges-from** - Higher level emerges from lower level
5. **Coordinates** - Coordinates activities of lower level
6. **Regulates** - Regulates behavior of lower level
7. **Composes** - Multiple elements compose to form higher level

### Pattern Holarchy Structure

```
Level 0: Source Foundation
├── Source (Unity ground)

Level 1: Dynamic Manifestation  
├── Dynamics (Temporal change)
└── Rhythm (Temporal cycles)

Level 2: Creative Polarity
├── Creative (Innovation)
└── Polarity (Distinction)

Level 3: Structural Exchange
├── Structure (Organization)

```

## Usage Examples

### TypeScript Usage

```typescript
import {
  CORE_PATTERNS,
  getPatternAspect,
  evaluateRelation,
  composePatterns
} from './lib/pattern-relational.types';

import {
  PATTERN_MATRIX,
  GLOBAL_PROCESSES,
  META_PATTERNS
} from './lib/pattern-relational.data';

// Get a specific relationship
const sourceToDynamics = PATTERN_MATRIX.source.dynamics;
console.log(sourceToDynamics.relationship); // "Unity Dynamics"
console.log(sourceToDynamics.cellNumber);   // 40

// Get pattern aspect
const aspect = getPatternAspect('source', 'rhythm');
console.log(aspect?.aspectName);  // "Field Vibrations"
console.log(aspect?.quality);     // "Unity Existence"

// Evaluate relational expression
const expr = evaluateRelation('source', '⊂', 'dynamics');
// "source ⊂ dynamics → Creative Grace"

// Compose patterns
const composed = composePatterns('dynamics', 'creative');
console.log(composed.description); 
// "Dynamic Creativity: Operation Dynamics"

// Find global process
const process41 = GLOBAL_PROCESSES.find(p => p.number === 41);
console.log(process41?.mechanism); 
// "limit cycle, eigen-behaviour, autopoiesis"

// Access meta-patterns
const unityField = META_PATTERNS.find(m => m.name === 'Unity Field');
console.log(unityField?.composedFrom); // ['source', 'rhythm']
```

### Query Pattern Relationships

```typescript
import { getAllPatternRelationships } from './lib/pattern-relational.types';

// Get all relationships for a pattern
const sourceRelationships = getAllPatternRelationships('source');
sourceRelationships.forEach(rel => {
  console.log(`${rel.pattern} → ${rel.column}: ${rel.aspectName}`);
});
```

### Navigate Holarchy

```typescript
import { HOLARCHICAL_RELATIONS } from './lib/pattern-relational.data';

// Find emergence relationships
const emergences = HOLARCHICAL_RELATIONS.filter(
  r => r.type === 'emerges-from' && r.child === 'source'
);

emergences.forEach(rel => {
  console.log(`${rel.parent} emerges from ${rel.child}: ${rel.description}`);
});
```
